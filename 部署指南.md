# 🚀 Pathivo 官网部署指南

## 项目说明

本网站使用 **Vite + React** 构建，通过 **GitHub Actions** 自动构建并部署到 **GitHub Pages**。

- 线上地址：https://yioyiu.github.io/Pathivo-Canvas-Demo/
- 部署方式：推送到 `main` 分支即可自动部署

---

## 一键部署（推荐）

只要将代码推送到 GitHub 的 `main` 分支，GitHub Actions 会自动完成构建和部署。

### 方式一：VS Code 内操作

```bash
# 1. 查看当前修改状态
git status

# 2. 添加所有修改的文件
git add .

# 3. 提交修改
git commit -m "feat: add Bing site verification and fix site URL"

# 4. 推送到 GitHub（会自动触发部署）
git push origin main
```

### 方式二：使用 VS Code 图形界面

1. 打开左侧 **「源代码管理」**（Ctrl+Shift+G）
2. 在输入框中写提交信息，例如 `feat: add Bing site verification`
3. 点击 **✔ 提交（Commit）**
4. 点击 **⇧ 同步更改（Sync / Push）**

---

## 查看部署进度

推送后，可以在 GitHub 上查看部署状态：

1. 打开 https://github.com/yioyiu/Pathivo-Canvas-Demo
2. 点击顶部的 **「Actions」** 选项卡
3. 你会看到正在运行的 `Build and Deploy to Pages` 工作流
4. 等待几分钟，绿色 ✓ 表示部署成功

---

## 验证部署结果

部署完成后，刷新网站确认：

- ✅ 打开 https://yioyiu.github.io/Pathivo-Canvas-Demo/
- ✅ 右键查看网页源代码，确认存在以下内容：

```html
<!-- Bing Webmaster Tools 验证 -->
<meta name="msvalidate.01" content="354850D53A68928ACC65F26D976EF134" />
```

## 本地构建测试（可选）

推送前可以在本地先构建测试，确保没有错误：

```bash
# 安装依赖
npm install

# 构建
npm run build

# 本地预览构建结果
npm run preview
```

## 部署后：Bing Webmaster Tools 验证

部署完成后，进行 Bing 的验证：

1. 访问 [Bing Webmaster Tools](https://www.bing.com/webmasters/)
2. 用 Microsoft 账号登录
3. **添加网站** → 输入 `https://yioyiu.github.io/Pathivo-Canvas-Demo/`
4. 验证方式选择 **「HTML Meta Tag」**
5. 系统会自动检测到我们已添加的 meta 标签，点击验证即可通过
6. 验证通过后，在左侧菜单 **「Sitemaps」** 提交：
   ```
   https://yioyiu.github.io/Pathivo-Canvas-Demo/sitemap.xml
   ```

## 常见问题

**Q: 推送后网站没更新？**
A: GitHub Actions 部署大约需要 1-3 分钟，可以到 Actions 页面查看进度。

**Q: 部署失败怎么排查？**
A: 到 GitHub 仓库的 Actions 页面点开失败的工作流，查看红色报错信息。

**Q: 修改后不想推送 main 分支？**
A: 可以创建一个新分支测试，确认没问题再合并到 main：
```bash
git checkout -b test-branch
# ...修改代码...
git add .
git commit -m "test changes"
git push origin test-branch
# 之后在 GitHub 上创建 PR 合并到 main
```
